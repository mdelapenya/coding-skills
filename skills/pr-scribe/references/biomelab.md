# Biomelab Integration

When pr-scribe detects a `.biomelab/` directory in the repository root, it activates a specialized workflow for the biomelab GUI application.

## Directory Structure

```
.biomelab/
├── topic.md        # Topic file with biomelab-specific instructions and conventions
├── pr-title.md     # Output: generated PR title (written by pr-scribe)
└── note.md         # Output: generated PR description (written by pr-scribe)
```

## Topic File (`.biomelab/topic.md`)

The topic file contains instructions and conventions for the biomelab context. It guides the pr-scribe skill in generating outputs that match the project's expectations.

### Expected Content

The topic file should include:
- **Naming conventions**: How to format the PR title (e.g., scope prefixes, required keywords)
- **Description structure**: Template or guidelines for the PR description format
- **Project-specific context**: Any custom fields, required sections, or special instructions

### Example

```markdown
# Biomelab PR Conventions

## Title Format
Use Conventional Commits with the `biomelab` scope for internal changes:
- `feat(biomelab): <description>`
- `fix(biomelab): <description>`
- `docs(biomelab): <description>`

For changes affecting external integrations, use the integration name:
- `feat(docker-integration): <description>`

## Description Structure
- **What's changed**: 1-2 sentences summarizing the core change
- **Why this matters**: Impact on biomelab workflows
- **Testing**: How to verify the change
- **Breaking changes**: If applicable, explain migration steps
```

## Output Files

### `pr-title.md`

Contains the single-line Conventional Commits title generated by pr-scribe. This is read by the biomelab GUI to populate the PR title field.

### `note.md`

Contains the full PR description markdown generated by pr-scribe. This is read by the biomelab GUI to populate the PR body field.

## Workflow

When pr-scribe detects the `.biomelab/` directory:

1. Load `.biomelab/topic.md` to understand the project's conventions
2. Detect the git platform (GitHub, GitLab) from `git remote -v`
3. Fetch PR metadata and diff (same as standard workflow)
4. Generate the PR title and description, applying any conventions from the topic file
5. Write outputs to:
   - `.biomelab/pr-title.md` — the title
   - `.biomelab/note.md` — the description
6. Skip the standard platform-specific update step (do not run `gh` or `glab`)
7. Inform the user that files have been written for the GUI app to consume

## Integration with biomelab GUI

The biomelab application reads the output files from `.biomelab/` and uses them to:
- Pre-populate PR title and description fields in its UI
- Allow the user to review, modify, and send the PR through the GUI's own mechanisms
- Track PR state and metadata separately from the git hosting platform

This allows pr-scribe to focus on analysis and description generation while the GUI handles actual PR creation/updates.
